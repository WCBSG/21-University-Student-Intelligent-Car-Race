# WCBSG 关键参数

更新时间：2026-07-16

本文件记录实车标定结果和当前单车 N=1 验收配置。运行时配置以
`CODE/config.json` 为准；`CODE/.flash` 由构建脚本生成，不要手工修改。

## 1. 电机与坐标约定

- 三轮指令 `[+s, +s, +s]`：车体顺时针旋转，绝对 Yaw 减小。
- `MotionControl.move(s, 90)`：车体左移。
- `MotionControl.move(s, -90)`：车体右移。
- `MotionControl.move(s, 0)`：车体前进，轮速约为 `[+s, -s, 0]`。
- 航向执行极性：`航向执行极性 = -1.0`。
- 视觉执行极性：`视觉执行极性 = +1.0`。
- ORBIT 侧移方向初值：`绕行方向 = +1.0`，仍需实车动态验证。

## 2. IMU 与绝对航向

- 左下角发车区车头正对场心：`Yaw = 0.0°`。
- 当前场心航向：`场心航向 = 0.0°`。
- 静止短期波动：约 `±0.1°`。
- 磁航向相对误差 `mrel` 实测约在 `-1.1° ~ +1.2°`。
- 磁力计开关：开启。
- 硬铁偏移：
  - X：`1634`
  - Y：`-182`
  - Z：`-662`
- 电机运行时当前航向来源以陀螺积分为主；静止时再使用磁力计缓慢校正。

## 3. 比赛与目标

- 单车验收目标数：`N = 1`。
- 发车位置：`2`，表示左下角。
- 严格目标：开启。
- 单车目标类别：`1`，网球。
- 搜索顺序：`[1, 0, 2]`，网球、沙袋、熊。
- 类别定义：
  - `0`：沙袋
  - `1`：网球
  - `2`：熊
- 推送航向偏角 `[沙袋, 网球, 熊]`：`[+90°, 0°, -90°]`。
- 当前网球推送目标航向：`0°`。

## 4. 相机和距离阈值

- 视觉水平中心目标：`cx = 50`。
- 本次绕行距离样本：
  - `y2 = 78.4`
  - `area = 597.9`
  - `score = 31`
  - 当时 `cx ≈ 65` 是目标未居中，不是摄像头中心偏移。
- ORBIT 起始阈值：`绕行起始位置 = 75`。
  - 低于实测理想值 78.4，为车辆减速和刹车保留余量。
- 推杆距网球约 1 cm 时：
  - `y2 = 99.6`
  - `cx ≈ 50.8`
  - `area ≈ 819 ~ 961`
- FINAL_APPROACH 视觉移交阈值：`接触位置 = 94`。
  - 该值表示从视觉接近切换为航向锁定直推，不代表推杆已经物理接触。
  - 暂不设置为 99.6，避免目标贴近图像边缘时先丢失。
- 预赛直接接近阈值：`停止位置 = 95`。
- 最低置信度：`18/31`。
- 确认帧数：`4`。
- 丢失帧数：`4`。

## 5. ORBIT 参数

- 绕行速度：`10%`。
- 绕行方向：`+1.0`，待首次动态测试确认。
- 径向比例：`0.6`。
- 径向输出上限：`8%`。
- 超时：`8000 ms`。
- 航向容差：`±8°`。
- 视觉居中容差：`cx = 50 ± 8`。
- 距离容差：`y2 = 75 ± 7`。
- 完成确认：连续 `4` 个新相机帧。
- 允许连续丢失：`6` 个新相机帧。
- 最终接近速度：`12%`。
- 最终接近超时：`5000 ms`。
- ORBIT 与 FINAL_APPROACH 总硬超时：`15000 ms`。

ORBIT 首次验收标准：

- Yaw 误差总体持续减小，不能连续发散。
- `cx` 最终进入 `42 ~ 58`。
- `y2` 最终进入 `68 ~ 82`。
- 上述条件与航向条件同时满足四帧后进入 `FINAL_APPROACH`。

## 6. TCS3472 黄线标定

普通蓝色地面实测：

- `R ≈ 367 ~ 378`
- `G ≈ 603 ~ 630`
- `B ≈ 842 ~ 869`
- `C ≈ 1902 ~ 1974`
- `rn ≈ 0.19`
- `gn ≈ 0.32`
- `bn ≈ 0.44`

黄色胶带实测：

- `R ≈ 2118`
- `G ≈ 2110`
- `B ≈ 836`
- `C ≈ 5475`
- `rn ≈ 0.39`
- `gn ≈ 0.39`
- `bn ≈ 0.15`

当前判定阈值：

- `rn >= 0.28`
- `gn >= 0.28`
- `bn <= 0.25`
- `C >= 800`
- 连续确认帧数：`4`

两类数据分离度充足，当前无需调整阈值。

## 7. 速度与超时

当前活动配置：

- 行驶占空比：`55%`
- 推箱占空比：`75%`
- TRACK 接近速度：`20%`
- FINAL_APPROACH 速度：`12%`
- SEARCH 速度：`15%`
- 行驶超时：`4000 ms`
- 推箱超时：`10000 ms`
- 黄线后继续清线：`500 ms`
- 掉头超时：`1500 ms`
- HOME 总超时：`12000 ms`
- 通用航向容差：`±12°`

首次完整动态测试应使用架空车轮或足够大的急停区域。若机械冲击明显，应先降低
`行驶占空比` 和 `推箱占空比`，不要通过放宽状态阈值掩盖速度问题。

## 8. 左下角 HOME 初始路线

当前回库策略为“绝对 Yaw 粗导航 + 黄线上升沿确认”：

1. 若计分后仍压黄线，先倒车离线。
2. 第一段目标航向：`+90°`。
3. 直行至第一次黄线上升沿。
4. 后退离线。
5. 第二段目标航向：`+135°`。
6. 直行至第二次黄线上升沿。
7. 制动并进入 `DONE`。

`+90°`、`+135°` 是当前几何初值，尚未经过完整回库实测。HOME 超时必须进入
`FAULT`，不能在未确认回库时伪报 `DONE`。

## 9. 调试输出

`标定输出 = true` 时每 500 ms 输出：

- `[CAL]`：Yaw、磁航向、磁力计原始值、目标类别、置信度、cx、y2、area。
- `[NAV]`：比赛阶段、目标 Yaw、Yaw 误差、cx、y2、前后/侧移/旋转命令、
  三轮实际 PWM、MotorArbiter owner、ORBIT 确认帧和丢失帧。
- `[TCS]`：RGBC、归一化通道和黄线判定。

正式比赛前将 `标定输出` 改为 `false`。
