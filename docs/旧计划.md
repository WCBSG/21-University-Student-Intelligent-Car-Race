## 接口契约（锁定）

```text
Config:     cfg.heading / cfg.tracking_bearing / cfg.tracking.*
            引用模式，Controller 不缓存副本，改字段即生效
Arbiter:    acquire(id) / release(id) / write(id, duties) / force_brake()
Intent:     ABORT|STOP|START_TRACK|GO_STRAIGHT|LOCK_YAW|RECONNECT
            每拍 drain 全部；含 ABORT → 只执行 ABORT，清空其余
FSM:        robot.handle(intent, arg=None) → bool
            robot.tick(dt, sensors) → None
            robot.on_camera_frame(has_target, y2=0.0)  # 或 tick 内 new_frame 触发
            sensors = {
              new_frame, has_target, target, y2, cam_timeout
            }
Mode:       enter() / update(dt, sensors) → None / exit()
            不自行转移；owner id = 状态名（HDG/SEARCH/TRACK/…）
CameraRx:   poll() → DetectionFrame | None
            timed_out → bool
            空帧 DetectionFrame(num=0) 清目标
RECONNECT:  robot.reconnect_pending；仅 IDLE|FAULT 可发起；成功→IDLE
build_robot(arbiter, cfg, imu) → RobotFSM（注册全部 Mode）
```

## 二、详细实现计划（原 最终计划.md）

# 单车完赛全流程 · 最终计划（详细版）

> 状态：**已定案**（Grok ↔ DS 对齐，2026-07-15）  
> 目标：一辆车单独跑通「发车 → 推完 N 件 → 回发车区」  
> 正式赛规要求双车协同计分；本计划为**队内里程碑 / 预赛向实现**，双车后置。  
> 规则摘要见 [MatchRule.md](MatchRule.md)；协作记录见 `.claude/handoff/`。

---

## 1. KPI

```text
摆车（车头朝场心，航向初始化 ≈ 0°）
  → 出库直行直到看见目标
  → 循环：锁定 →（决赛转向）→ 推出黄线 → 掉头回中心再搜
  → 满 N 件后：两段回库 → DONE
```

- **N** = `cfg.match_target_count`（默认 3，赛前可改）。  
- **预赛**：四面出界均可 → 可跳过 PRE_PUSH。  
- **决赛**：网球→上、沙袋→左、熊→右 → 必须 PRE_PUSH 方向映射（与预赛间隔约 24h，必须做）。

---

## 2. 状态机

```text
LEAVE → PICK → APPROACH → PRE_PUSH(决赛) → PUSH → SCORE
         ↑                                          │
         └──── 转180° + 直行直到见目标 ──────── scored < N
                                                 scored ≥ N → HOME → DONE
```

| 阶段 | 做什么 | 退出条件 |
|------|--------|----------|
| **LEAVE** | `MotionControl.move(+duty, 0)`，航向锁 0° | 摄像头稳定发现目标 → PICK；或超时（如 5s）→ 停转搜进入 PICK |
| **PICK** | 复用 SEARCH；`target_class = _remaining[0]`（优先未搬类别） | SEARCH → TRACK |
| **APPROACH** | 复用 TRACK，bearing PI 接近 | TRACK → COMPLETE（y2≥阈值） |
| **PRE_PUSH** | **仅决赛**：旋转到 `H_ref + OFFSETS[layout][cls]` | 航向到位 → PUSH；预赛 / `layout==0` **跳过** |
| **PUSH** | Runner 接管 Arbiter，`move(+push_duty, 0)` 直推 | TCS **OFF→ON 上升沿** → SCORE；或推超时（如 3s）→ SCORE |
| **SCORE** | `scored_count += 1`；更新 `_remaining` | `< N` → 回中心再 PICK；`≥ N` → HOME |
| **回中心**（合在 SCORE 后，原 NEXT） | 转 ~180° → 直行至再发现目标 | → PICK + START_TRACK |
| **HOME** | **两段**（见 §5） | 第二段黄线↑ 或总超时 → DONE |

急停：ABORT + brake → phase IDLE（任意阶段）。

---

## 3. 运动学硬约束

本车三角全向轮（[CODE/Motor.py](CODE/Motor.py)）：

| 动作 | 正确指令 | 错误 |
|------|----------|------|
| 平移前进/后退 | `MotionControl.move(+s, 0)` / `move(-s, 0)` | 三轮同值 |
| 原地旋转 | `[s, s, s]` 或 `[-s,-s,-s]`（与 HDG lock 一致） | 用 `move` 假装转 180° |

LEAVE / PUSH / 回中心直行 / HOME 直行 → 一律 `move`。  
PRE_PUSH 转向、回中心掉头、HOME 转向 → 三轮同值（或短时航向闭环，实现自选，优先开环计时+到位阈值）。

## 4. 问题 1–5 定案摘要

### 4.1 出库（LEAVE）

- 摆车：**车头朝物体/场心**；IMU 标定完成后再发车。  
- 初始化航向视为 **0°**。  
- 直行直到视觉稳定见目标；**超时**则进入原地搜索，禁止假设「发车位转圈一定看得见中心」。

### 4.2 指定方向（PRE_PUSH）

- **磁力计不能单独认「场地上边」**（给的是地磁方位）。  
- **方向映射**：到场确认发车区位置 → 配 `start_layout`；摆车朝场心采 `push_hdg_ref`（`H_ref`）；  
  `target_yaw = wrap(H_ref + OFFSETS[layout][cls])` → 转到位再推。  
- 几何直觉（到场重标，勿写死）：  
  - 底边正中朝场心：`H_ref` ≈ 网球（上）方向；  
  - 左下角朝场心：`H_ref` ≈ 上与右交界 → 用 offset 推出三类。  
- `layout == 0` 或三偏角全 0 → **预赛直推**。

### 4.3 黄线与防重复

**TCS 状态机：**

- `OFF_LINE` → 连续 N 帧黄（如 N=4）→ `ON_LINE`（确认上线）。  
- `ON_LINE` → 连续 N 帧非黄 → `OFF_LINE`（确认离线）。  
- **仅**在 `OFF_LINE→ON_LINE` **上升沿**且当前 phase==**PUSH** 时计分一次。  
- 压在同一条线上连续见黄：**不重复计分**。

**推出后：**

- 转 ~180° → 直行（同 LEAVE）直到再发现目标 → PICK。  
- **规则允许**线外物体由人拿走（须等车不再搬该物）；SCORE/掉头后再拿。

### 4.4 绕障

- **本轮不做**（预赛无障碍）。  
- 决赛以后保底设想（未实现）：凸起可压；超时换下一目标；不承诺几何绕砖。

### 4.5 回库（HOME，两段）

与方向映射共用 `H_ref` + `layout`：

1. 若仍 `ON_LINE`：先短退至确认 `OFF_LINE`。  
2. **Leg1**：转到「靠库的物体侧边」航向（左下库→左边/沙袋边；右下库→右边/泰迪边）→ 直行 → 黄线上升沿。  
3. 短退离线。  
4. **Leg2**：转向发车区开口航向 → 直行 → 黄线上升沿（或超时）→ DONE。  
5. 若库在边正中（如左边中）：`yaw_leg2 ≈ yaw_leg1`，可跳过 Leg2。  
6. HOME 总超时兜底（强制 DONE，完赛优先）。

对齐航向时宜停车/低速，便于 mag 可用时纠漂；MATCH 默认 mag 策略以实现时 config 为准（静止窗口可开）。

## 5. Config 字段（建议）

| 字段 | 含义 | 默认建议 |
|------|------|----------|
| `match_target_count` | 要推件数 N | `3` |
| `start_layout` | `0` 预赛直推；`1` 底边中；`2` 左下；`3` 右下；（建议预留左边正中） | `0` |
| `push_hdg_ref` | 朝场心时 mag/航向参考 `H_ref` | `0.0`（到场填写） |
| `OFFSETS` 或等价 | `layout × cls → 偏角°` | 到场表 |
| `leave_duty` / `leave_timeout_ms` | 出库 | `15` / `5000` |
| `push_duty` / `push_timeout_ms` | 顶推 | `12` / `3000` |
| `tcs_confirm_frames` | 上/离线连续帧数 N | `4` |
| `home_timeout_ms` | 整段 HOME 上限 | `10000`～`15000` |
| `next_spin_ms` 等 | 掉头开环时间 | 实车标定 |

OpenART `cls_id` 必须与表中沙袋/网球/熊一致（实现前核对相机类别表）。

---

## 6. 赛前标定步骤

1. 车放发车区，**车头朝场心**，静止完成 IMU 陀螺零偏。  
2. 开 mag（若用），读航向 → 写入 `push_hdg_ref`。  
3. 选择/确认 `start_layout`，核对 OFFSETS（转车目视上/左/右是否与表一致）。  
4. 硬铁标定（换场地建议重做）。  
5. 设 `match_target_count`。  
6. 一键发车，之后不再改参（正式赛规则）。

## 8. 验收清单

1. **LEAVE**：朝场心开，见目标后进入搜索/跟踪；超时能转搜。  
2. **N=1**：推一件（上升沿只计 1）→ HOME 两段 → DONE；压线不秒结束。  
3. **N=2**：掉头回中心再锁第二件；线外物拿走后不重复锁同一逻辑件。  
4. **预赛** `layout=0`：无 PRE_PUSH，直推仍能出线。  
5. **决赛表**：改 OFFSETS 后推前明显转向。  
6. DEBUG Start Match / MATCH 自动均可跑通；急停回 IDLE。
