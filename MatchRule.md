## 1. 赛题定位

- 组别：**蚂蚁搬家** — MicroPython 主战场；车模开放自制；两辆步兵车计重计分，需考虑轻量化。
- 比赛约 **2026 年暑期**；正式文档以全国大学生智能车竞赛网站为准。
- 编程语言：**仅 Python（MicroPython）**，禁止 C 应用层程序；线下可能查固件 / 换现场核心板。

---

## 2. 基本任务（细则归纳）

1. 制作 **两辆搬运车模（步兵）**，按物品种类从场中央搬到场边（预赛）或指定边（决赛）。
2. 物品含网球、沙袋、泰迪熊等（任务说明中的三类；细则也曾泛提球/布偶等，以任务说明为准）。
3. **须两车（或以上）协同推动**；搬运过程中物体 **不离开地面**。
4. 参与搬运的车模之间 **禁止物理连接**；允许 **无线通信**。
5. 另可设 **≤2** 台辅助车模（工兵）或导引信标（目标发现、路径、发车区导航等）；辅助 MCU 用 **STC**。
6. 辅助车可动可不动；高 ≤50 cm；边长 ≤30×30 cm；**禁止触碰搬运物品**。
7. 含 MCU 的图像平台限定：**OpenART / OpenART Plus / MCXVision / MV5-RT**（NXP）。

## 3. 场地与流程

| 项 | 规定 |
|----|------|
| 正式场地（兼容 AI 视觉组） | **3.2 m × 2.4 m** |
| 背景 | 内外蓝色背景布（Q&A：多为深蓝，便于与浅蓝沙包区分） |
| 边界 | 宽 **5 cm 黄胶带**（颜色无统一采购标准） |
| 物品区 | 中心约 **1 m × 1 m**；物品间距约 **≥10 cm** |
| 摆放形态 | 演示/零裁判模式多为 **九宫格**（规格按官方演示视频；尺寸图不单独公布） |
| 发车区 | 靠 **左侧黄线** 一侧；围线黄或白；出入必须过 **场地与发车区重叠的前边**（计时门） |
| 发车朝向 | 无强制朝向，但须从面向赛场一侧出发 |

**赛前流程：** 搬运车进发车区 → 辅助车固定 → 再摆物品 → 之后 **禁止再操作车**，只允许 **按一次发车键**（现场可能有“盲盒”启动方式）。

**计时：** 两辆搬运车 **离开发车区进场开始** → **两车都回到发车区**，以 **最后一辆回库** 时刻结束。  
- 国赛要求 **两车都回库**（省赛可能自定，以当地规则为准）。  
- 回库触发计时即可，**不必停车**；触发后仍可再驶出。  
- **须搬完所有物品后再回库**；中途回库会被计时系统视为结束。

**成功判定：**

- 物品 **离开黄线** 即成功（车可仍在线内）。
- 成功后物品自行滚回线内仍算成功。
- **每次只搬一件**；一次推出多个 → 犯规 / 遗漏罚时。
- **不允许单车独立完成**官方有效搬运（须双车协同接触推动）。

## 4. 搬什么 / 搬到哪

### 物品

| 种类 | 外观 | 尺寸 |
|------|------|------|
| 标准网球 | 浅绿 | ⌀ 6.54–6.86 cm |
| 玩具沙袋 | 红 / 浅蓝（宜与背景可分） | 边长约 7 cm |
| 泰迪熊 | 棕 / 白 | 高度 &lt; 20 cm |

- 同类可出现 **多个**；数量预赛/决赛具体值赛前公布（曾提及预赛约 3–5 个量级，以现场为准）。
- **不可自带**比赛用物品。

### 目标区域

| 阶段 | 要求 |
|------|------|
| **预赛** | 推出场地外即可；**四边均可**，无方向限制 |
| **决赛** | 网球 → 上方；沙袋 → 左方；泰迪熊 → 右方 |
| 出界认定 | **第一次出界边** 视为到达区域（不可先出一边再场外挪到另一边算改判） |

预赛无障碍；决赛有凸起 + 砖块（华东等赛区安排以领队会为准，可能一上来即有障碍）。

## 5. 双车协同（Q&A 要点）

| 要点 | 结论 |
|------|------|
| 必须双车推 | 有效搬运须 **两车都与物体接触** 推动；长期仅单车推不合规 |
| 瞬间脱接触 | 允许短暂不同时接触；不允许全程单车推 |
| 分段搬运 | 可先单车微调再双车推出；可二段搬（网球滚开后再找） |
| 推出后 | 物品已出线后，可单车再往外推远一点 |
| 车身 | **禁止用车体搬运**；允许触碰但不能靠车身推运 |
| 拿走出界物 | 物品出线后可人为拿走，但须等 **两车不再搬该物** |
| 物理连接 | 禁止；允许无线 |

## 6. 车模与推杆

| 项 | 限制 |
|----|------|
| 外形 | 含推杆落在 **30×30 cm** 方形内 |
| 推杆数量 | **只能 1 根** |
| 推杆水平长度 | **≤ 10 cm** |
| 推杆竖直高度（杆截面方向 H） | **≤ 5 cm** |
| 推杆离地高度 | 规则未限，自定 |
| 形状 | 须为 **直杆 / 矩形推面**；禁止钩、叉、兜、铲、镂空成抓取结构等 |
| 挖空 | **推面不允许挖空**（可贴纸封住） |
| 海绵/砂纸 | 允许；海绵形变建议 **&lt;5 mm** |
| 与车夹角 | 不限；**动态改夹角**曾允许，后 Q&A 明确 **禁止舵机主动施力式机械臂类**（避免衍生机械臂）——以最新 Q&A/现场为准 |
| 轮侧护罩 | 允许（防撞轮/防碰物） |
| PCB | **必须自制**（驱动等）；禁止整板买成品拼装过关 |
| 重量罚时 | 超 **100 g** 起罚（见总规则前言）；按 **两车平均重量**；无上限；约超重相关罚时标准见总规则 |

辅助：**≤2**（辅助车与信标合计）；不可碰物品；发车后不可再人为挪辅助车（赛前摆好可调整一次类操作以 Q&A 为准）。

## 7. 障碍（决赛）

| 类型 | 说明 |
|------|------|
| 路面凸起 | 类信标外壳缓坡，约 ⌀25 cm、中心高约 1.5 cm；蓝布覆盖；可压可躲；在物品区外；数量约 3–5（无硬性范围） |
| 砖块 | 距边界约 **50 cm** 内；个数 **≤3**；普通红砖或现场模拟砖；不可移动；可有角度摆放 |

预赛：**无**上述障碍。

## 8. 硬件 / 编程硬约束

### 主控（步兵）

- 指定 **NXP RT1021 MicroPython 核心板**（特价 BTB 等）；**仅官方固件**。
- 开发：Thonny + Type-C；禁止 DAP/J-Link 下应用；禁止另加 NXP 协处理器跑 C。
- 固件模块概要：`machine`（Pin/ADC/UART/PWM）、`smartcar`（ticker/encoder 等）、`seekfree`（IMU660/963、电机、无线等）、`display`。

### 视觉 / 传感器

| 允许 | 限制 |
|------|------|
| OpenART 系列等指定视觉模组 | 带 MCU 的摄像头不可用其它品牌替代（卡丁/飞跃雷区等同限） |
| 无 MCU 的摄像头、光流、测距、普通 RGB 传感器成品 | 带 MCU 的激光雷达模组等一般不允许 |
| 车模间无线 | 允许（白名单无线模块可用） |
| UWB | 可；固定在车上；数量与车数相关（不宜乱放场外一堆） |
| 深度相机 | 有条件允许；**不可外挂树莓派** |

### 辅助车

- MCU：**STC**（不可用 RT1021 / MCX 等当辅车主控）。
- 辅车不可给主车供电（硬件盲盒相关问答）。

## 9. Q&A 速查（高频）

| 主题 | 结论 |
|------|------|
| 搬运顺序 | 不指定；队伍自选 |
| 回库时机 | 全部搬完再回；中途回库=结束 |
| 两车回库 | 国赛要两车都回；计最后一辆 |
| 一次多物出界 | 犯规/遗漏罚时 |
| 跟丢从车 | 算失败 |
| 物品自带 | 否 |
| 赛中改参 | 摆物后不允许；正式赛一键发车 |
| 颜色阈值 | 赛前调试可以；赛中摆物后不行 |
| 地图 | 初赛可同题测量；决赛随机抽取摆位 |
| 激光雷达（二维） | 细则文有“可以”；带 MCU 部件另有限制——以最新白名单+现场为准 |
| 光流 | 可以 |

|------|------|
| DEBUG | 菜单：IMU / Start Match / Tracker |

PICK → APPROACH → PUSH → SCORE → DONE
